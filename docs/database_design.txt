DATABASE DESIGN DOCUMENT (DDD)
1. Purpose
This document defines the complete database architecture for SOTS.
The document includes:
* Database technologies
* Data ownership
* Schema design
* Entity relationships
* Indexing strategy
* Partitioning strategy
* Retention strategy
* Scalability strategy
* Multi-tenancy strategy

Supports:
* Workflow discovery
* State relationships
* Transition analysis
* Test generation

Every database contains:

organization_id


where applicable.
Isolation:

    │
    ├── Events
    ├── Sessions
    └── Reports

    │
    ├── Events

    └── Reports
